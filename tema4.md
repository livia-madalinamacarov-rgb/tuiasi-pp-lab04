Problema 1
   
     In aceasta problema am facut implementarea unui Web Crawler utilizând biblioteca Jsoup, conform cerinței din laborator . Aplicația are rolul de a prelua conținut de la un URL și de a-l procesa în funcție de tipul acestuia( JSON, XML sau YAML).Am construit interfata Parser, care definește o metodă parse. Aceasta primește un text și returnează un Map (cheie-valoare).Toate tipurile de parsere trebuie să implementeze această metodă.Pentru fiecare tip de conținut există o clasă separată. XmlParser folosește Jsoup pentru a parsa XML-ul, ia elementul rădăcină și extrage pentru fiecare element copil numele tagului și textul. JsonParser și YamlParser sunt mai simple și doar returnează conținutul ca text, fără să îl proceseze efectiv. Clasa Crawler ia datele de la URL și alege parserul potrivit. În funcție de tipul dat (json, xml sau yaml), creează parserul corespunzător și procesează conținutul, iar rezultatul este apoi afișat. Aplicația respectă principiile SOLID. De exemplu, clasa Crawler nu depinde direct de clasele concrete, ci folosește interfața Parser, ceea ce face codul mai usor de modificat. De asemenea, se pot adăuga ușor noi tipuri de parsere fără să fie nevoie să modific codul existent.De asemenea, am comentat si codul pentru explicatii suplimentare.

Problema 2
   In aceasta problema am realizat o aplicatie  simpla pentru cumpararea biletelor la cinema(limbajul Kotlin).Programul este format din mai multe clase. Clasa Movie descrie un film prin titlu si pretul biletului. Clasa Screening reprezinta proiectia filmului la o anumita ora. Clasa Client contine informatii despre client, iar clasa Seat informatii despre un loc(rezervarea lui, daca este deja ocupat sau nu). Am folosit si interfata PaymentMethod care stabileste metoda de platasi este implementata de clasele CashPayment si CardPayment. Daca se foloseste metoda de plata cu cardul, se utilizeaza clasa BankAccount, care contine informartii despre datele cardului. Clasa Cinema se ocupă  de achiziția biletelor, prin metoda cumparaBilet (verifică daca locul este disponibil  și realizeaza plata, iar daca  ambele condiții sunt îndeplinite, creează și returnează un obiect de tip Ticket care contine informatii despre client, film, ora, loc si pret).Sunt respectate, pe cat posibil principiile SOLID. Single Responsibility(fiecare clasa are un rol clar), Open/Closed(interfata PaymentMethod, care permite adaugarea unei metode noi farsa sa modifice codul), Liskov Substitution(CashPayment, CardPayment pot fi folosite oriunde  este necesar un PaymentMethod), Interface Segregation(interfata simpla), Dependency Inversion(clasa cinema depinde de metode abstracte, nu de implementari concrete).

Problema 3

    In aceasta problema am realizat o aplicatie care reprezinta un sistem simplu de gestionare a notitelor(limbajul Kotlin). Dupa cum era scris si in cerinta, aplicatia permite celui care o foloseste sa creeze, sa incarce, sa caute si sa stearga notite, fiecare fiind  separata pe disk.Programul este alcatuit in mai multe clase. Clasa utilizator, care reprezinta utilizatorul clasei si clasa Notita(contine informatii despre o notita).Am definit si interfata RepositoryNotite, care implemneteaza metode precum salvarea unei notite,obtinerea tuturor notitelor, cautarea unei notite dupa id si stergerea lor.Clasa ManagerNotite implementeaza interfata. Cand se creeaza un obiect ManagerNotite, se verifica daca exista un folder "notes", iar daca nu exista, sa creeaza automat. De asemenea, cand se creeaza o notita, aplicatia creeaza un id unic(cel mai mare id existent, la care se adauga 1). Fiecare notita este salvata intr un fisier separat(numele este dat de id si extenstia .txt). Pentru data și ora, folosesc clase din pachetul java.time. Data este obținută folosind LocalDate.now()(returnează data în format standard (YYYY-MM-DD)). Ora  este obținută cu LocalTime.now(), dupa  este formatată folosind DateTimeFormatter,  astfel încât ora să fie afișată în formatul ore:minut:secundă.(HH:mm:ss). De asemenea, am comentat si codul pentru explicatii suplimentare.
